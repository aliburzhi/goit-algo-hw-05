# Порівняння алгоритмів пошуку підрядка

## Опис завдання

Порівняння ефективності трьох алгоритмів пошуку підрядка:

- Боєра-Мура 
- Кнута-Морріса-Пратта 
- Рабіна-Карпа
Вимірювання проводились за допомогою `timeit` (1000 повторень) на двох текстових файлах

| Стаття 1 | Вигаданий | `квантовий нейромережевий блокчейн` |

| Стаття 2 | Вигаданий | `гіперпросторовий індексатор даних` |

## Висновки

### 1. Боєра-Мура — найшвидший

Алгоритм Боєра-Мура виявився найшвидшим в усіх чотирьох тестах. Особливо помітна перевага при пошуку вигаданих підрядків: завдяки правилу «поганого символу» алгоритм може пропускати великі ділянки тексту, не виконуючи зайвих порівнянь. На довгих підрядках, яких немає в тексті, виграш над Кнута-Морріса-Пратта досягає 9–7×.

### 2. КМП — середній результат

Алгоритм Кнута-Морріса-Пратта посів друге місце. Він стабільно працює за лінійний час O(n + m) і не має «поганих» випадків, але не вміє робити великих стрибків по тексту, як Боєра-Мура. Практично Кнута-Морріса-Пратта краще підходить для текстів з великою кількістю повторюваних символів (наприклад, бінарні дані), де Боєра-Мура втрачає перевагу.

### 3. Рабін-Карп — найповільніший

Алгоритм Рабіна-Карпа показав найгірші результати для однорядкового пошуку. Це пояснюється тим, що його сильна сторона — пошук кількох шаблонів одночасно (хешування дозволяє перевіряти багато підрядків за один прохід), тоді як для пошуку одного підрядка накладні витрати на обчислення хешів додають постійний оверхед.

### 4. Вигадані підрядки vs існуючі

В усіх алгоритмах пошук вигаданого підрядка займав більше часу, ніж існуючого — адже при невдалому пошуку алгоритм змушений переглянути весь текст до кінця. Боєра-Мура і тут виграє: стрибки по таблиці зміщень скорочують кількість порівнянь навіть при повному переборі.

## Рекомендації щодо вибору алгоритму

| Сценарій | Рекомендований алгоритм |
|----------|------------------------|
| Загальний пошук у тексті | Боєра-Мура |
| Текст з повторюваними символами (ДНК, бінарні дані) | Кнута-Морріса-Пратта |
| Пошук кількох шаблонів одночасно | Рабін-Карп |
| Потоковий пошук (текст надходить поступово) | Кнута-Морріса-Пратта |
